# CI/CD Auto-Publish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up a Flutter project with GitHub Actions CI/CD that auto-deploys Web to GitHub Pages and Android debug APK to GitHub Releases.

**Architecture:** Single Flutter project (default Counter App) with one GitHub Actions workflow containing 3 jobs: build-web, build-android (tag-only), and deploy. Triggered by push tag `v*` or manual workflow_dispatch.

**Tech Stack:** Flutter 3.41.5, GitHub Actions, `subosito/flutter-action@v2`, `peaceiris/actions-gh-pages@v3`, `softprops/action-gh-release@v2`

**Spec:** `docs/superpowers/specs/2026-03-19-cicd-auto-publish-design.md`

---

## File Structure

| File | Action | Responsibility |
|------|--------|---------------|
| `.fvmrc` | Create | Pin Flutter version for FVM |
| `.gitignore` | Create | Ignore Flutter/Dart build artifacts and IDE files |
| `lib/main.dart` | Create | Default Counter App (generated by `flutter create`) |
| `pubspec.yaml` | Create | Flutter project config (generated by `flutter create`) |
| `web/` | Create | Web platform files (generated by `flutter create`) |
| `android/` | Create | Android platform files (generated by `flutter create`) |
| `.github/workflows/ci.yml` | Create | CI/CD workflow with 3 jobs |

### Task 1: Scaffold Flutter Project

**Files:**
- Create: `.fvmrc`, `pubspec.yaml`, `lib/main.dart`, `web/`, `android/`, `.gitignore` (all generated)

- [ ] **Step 1: Pin Flutter version with FVM**

```bash
fvm use 3.41.5
```

This creates `.fvmrc` in the project root. Verify the format:

```bash
cat .fvmrc
# Expected: {"flutter": "3.41.5"}
```

- [ ] **Step 2: Scaffold the Flutter project**

```bash
fvm flutter create . --project-name mini_play --platforms web,android
```

This generates the default Counter App with web and android platform support. The `--platforms` flag ensures only the needed platforms are created.

- [ ] **Step 3: Verify the project builds locally**

```bash
fvm flutter pub get
```

Expected: Dependencies resolve successfully, no errors.

- [ ] **Step 4: Update .gitignore**

Ensure `.gitignore` includes FVM-specific entries. Add to the existing generated `.gitignore`:

```
# FVM
.fvm/
```

- [ ] **Step 5: Commit**

```bash
git add .fvmrc .gitignore lib/ web/ android/ pubspec.yaml pubspec.lock analysis_options.yaml test/
git commit -m "feat: scaffold Flutter project with web and android platforms"
```

Note: Do NOT add `.fvm/` directory — it's local only.

### Task 2: Create GitHub Actions Workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/ci.yml` with the following content:

```yaml
name: CI/CD

on:
  push:
    tags:
      - 'v*'
  workflow_dispatch:

permissions:
  contents: write

jobs:
  build-web:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Read Flutter version from .fvmrc
        id: fvm
        run: |
          FLUTTER_VERSION=$(jq -r '.flutter' .fvmrc)
          echo "version=$FLUTTER_VERSION" >> $GITHUB_OUTPUT

      - uses: subosito/flutter-action@v2
        with:
          flutter-version: ${{ steps.fvm.outputs.version }}

      - run: flutter pub get
      - run: flutter build web --release --base-href "/mini-play/"

      - uses: actions/upload-artifact@v4
        with:
          name: web-build
          path: build/web

  build-android:
    if: startsWith(github.ref, 'refs/tags/v')
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Read Flutter version from .fvmrc
        id: fvm
        run: |
          FLUTTER_VERSION=$(jq -r '.flutter' .fvmrc)
          echo "version=$FLUTTER_VERSION" >> $GITHUB_OUTPUT

      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '17'

      - uses: subosito/flutter-action@v2
        with:
          flutter-version: ${{ steps.fvm.outputs.version }}

      - run: flutter pub get
      - run: flutter build apk --debug

      - name: Rename APK
        run: mv build/app/outputs/flutter-apk/app-debug.apk build/app/outputs/flutter-apk/mini-play-debug.apk

      - uses: actions/upload-artifact@v4
        with:
          name: android-apk
          path: build/app/outputs/flutter-apk/mini-play-debug.apk

  deploy:
    needs: [build-web, build-android]
    # Run even when build-android is skipped (manual trigger), but not if any job failed
    if: always() && !failure() && !cancelled()
    runs-on: ubuntu-latest
    steps:
      - name: Download web build
        uses: actions/download-artifact@v4
        with:
          name: web-build
          path: web-build

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./web-build

      - name: Download Android APK
        if: startsWith(github.ref, 'refs/tags/v')
        uses: actions/download-artifact@v4
        with:
          name: android-apk
          path: android-apk

      - name: Create GitHub Release
        if: startsWith(github.ref, 'refs/tags/v')
        uses: softprops/action-gh-release@v2
        with:
          generate_release_notes: true
          files: android-apk/mini-play-debug.apk
```

- [ ] **Step 2: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add CI/CD workflow for web and android deployment"
```

### Task 3: Push and Validate with Manual Trigger

- [ ] **Step 1: Push to remote**

```bash
git push origin main
```

- [ ] **Step 2: Trigger workflow manually**

Go to GitHub repo > Actions > CI/CD > Run workflow (or use CLI):

```bash
gh workflow run ci.yml
```

- [ ] **Step 3: Monitor the workflow run**

```bash
gh run list --limit 1
gh run watch
```

Expected: `build-web` succeeds, `build-android` is skipped, `deploy` succeeds (deploys Web to `gh-pages` branch).

- [ ] **Step 4: Enable GitHub Pages**

Go to repo Settings > Pages > Source: "Deploy from a branch" > Branch: `gh-pages` / `/ (root)`.

- [ ] **Step 5: Verify Web deployment**

Visit `https://cecil-su.github.io/mini-play/` — should show the Flutter Counter App.

### Task 4: Validate Tag Trigger (Full Pipeline)

- [ ] **Step 1: Create and push a tag**

```bash

```

- [ ] **Step 2: Monitor the workflow run**

```bash

gh run watch
```

Expected: `build-web` and `build-android` run in parallel, both succeed, `deploy` creates GitHub Release with APK attached.

Expected: Release `v0.0.1` exists with `mini-play-debug.apk` attached and auto-generated release notes.

- [ ] **Step 4: Verify Web is still accessible**

Visit `https://cecil-su.github.io/mini-play/` — should still show the Flutter Counter App (re-deployed).

- [ ] **Step 5: Commit any fixes if needed**

If any workflow issues were found and fixed during validation, commit the fixes.
