# 2048 Game Design Spec

## Overview

2048 is the second game in the mini-play collection. A classic sliding tile puzzle where players merge numbered tiles on a grid to reach 2048 and beyond.

## Modes

Four board sizes, selectable from a mode page:

| Mode | Grid | Description |
|------|------|-------------|
| 3×3 | 3×3 | 快速挑战 |
| 4×4 | 4×4 | 经典模式 |
| 5×5 | 5×5 | 扩展棋盘 |
| 6×6 | 6×6 | 终极挑战 |

## Architecture

### No Flame Engine

2048 is a turn-based grid game with no real-time physics or continuous rendering. Pure Flutter Widgets with AnimationController provide finer animation control and simpler code than Flame.

### File Structure

```
lib/game_2048/
├── game_2048_mode_page.dart      # Mode selection (3×3/4×4/5×5/6×6)
├── game_2048_page.dart           # Game page (StatefulWidget, receives gridSize)
├── game_2048_board.dart          # Board core logic (state, move, merge)
├── game_2048_tile.dart           # Tile data model
├── game_2048_colors.dart         # Classic color mapping
└── game_2048_animated_tile.dart  # Tile animation Widget
```

### Routing

- `/2048` → `Game2048ModePage`
- `/2048/play` → `Game2048Page` (receives `gridSize` via route arguments)

### Registry Updates

In `game_registry.dart`: set `implemented: true`, `hasModePage: true` for the 2048 entry.

Add routes in `main.dart`.

## Core Game Logic

### Board State

- 2D array `List<List<int>>` storing cell values (0 = empty).
- New tile generation: 90% chance of 2, 10% chance of 4.

### Slide Algorithm (e.g., slide left)

1. Process each row: remove zeros, merge adjacent equal numbers, pad with zeros.
2. Accumulate score from merges.
3. Only spawn a new tile if the board state changed.

### Undo System

- History stack, max 3 entries (board state + score snapshot).
- Push current state before each valid move.
- Undo pops the top entry and restores state; undo count decrements.
- 3 undo chances per game, non-replenishable.

### Game Over Detection

- No empty cells AND no adjacent equal cells in any direction → game over.

### Reaching 2048

- Show congratulations dialog with two options: "Continue" (endless mode) or "View Score".
- In endless mode, game continues with no upper limit on tile values.

### Score System

- Each merge adds the merged value (two 16s merge → +32).
- High scores persisted via `ScoreService` with key format: `highscore_2048_4x4`.

## Animations

All animations use Flutter's `AnimationController`, ~200ms duration.

| Animation | Effect |
|-----------|--------|
| Slide | Tween position interpolation from origin to target |
| Merge | Scale up to 1.2× then bounce back (ScaleTransition) |
| New tile | Scale from 0 to 1 + fade in (ScaleTransition + FadeTransition) |

- All animations run in parallel.
- Input is locked during animation to prevent state corruption.

## Input Handling

- **Keyboard**: Arrow keys + WASD.
- **Swipe**: GestureDetector onPanEnd, direction determined by velocity vector.
- **Animation lock**: Ignore input while animations are in progress.

## UI Layout

- Reuse `GameScaffold` for title, score display, pause functionality.
- Board centered with `AspectRatio(1.0)` to maintain square shape.
- Board background: rounded rectangle `#bbada0`, cell spacing adapts to gridSize.
- Undo button below the board, showing remaining undo count.

## Mode Selection Page

Follows `snake_mode_page.dart` pattern:

- Four mode cards with grid icon + name + description.
- Tap navigates to `/2048/play` with `gridSize` argument.

## Game Over Flow

1. **Reach 2048** → Dialog: "Congratulations!", buttons: "Continue" / "View Score".
2. **No moves left** → Navigate to `GameOverPage` with stats:
   - Score
   - Max tile value
   - Move count
   - Mode (e.g., "4×4")
3. `GameOverPage` provides "Play Again" and "Home" buttons (reused from shared).

## Shared Infrastructure Reuse

- `GameScaffold`: top bar, score, pause overlay.
- `ScoreService`: high score persistence.
- `GameOverPage`: game over screen with stats and replay.
