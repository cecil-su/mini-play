# 2048 Game Design Spec

## Overview

2048 is the second game in the mini-play collection. A classic sliding tile puzzle where players merge numbered tiles on a grid to reach 2048 and beyond.

## Modes

Four board sizes, selectable from a mode page:

| Mode | Grid | Description |
|------|------|-------------|
| 3×3 | 3×3 | 快速挑战 |
| 4×4 | 4×4 | 经典模式 |
| 5×5 | 5×5 | 扩展棋盘 |
| 6×6 | 6×6 | 终极挑战 |

## Architecture

### No Flame Engine

2048 is a turn-based grid game with no real-time physics or continuous rendering. Pure Flutter Widgets with AnimationController provide finer animation control and simpler code than Flame.

### File Structure

```
lib/game_2048/
├── game_2048_mode_page.dart      # Mode selection (3×3/4×4/5×5/6×6)
├── game_2048_page.dart           # Game page (StatefulWidget, receives gridSize)
├── game_2048_board.dart          # Board core logic (state, move, merge)
├── game_2048_tile.dart           # Tile data model
├── game_2048_colors.dart         # Classic color mapping
└── game_2048_animated_tile.dart  # Tile animation Widget
```

### Routing

- `/2048` → `Game2048ModePage`
- `/2048/play` → `Game2048Page` (receives `gridSize` via route arguments)

This diverges from Snake's per-mode routes (`/snake/classic`, etc.) because all 2048 modes share identical game logic differing only in grid size. Route argument extraction in `main.dart`:

```dart
case '/2048/play':
  final gridSize = settings.arguments as int? ?? 4;
  return MaterialPageRoute(builder: (_) => Game2048Page(gridSize: gridSize));
```

### Registry Updates

In `game_registry.dart`: set `implemented: true`, `hasModePage: true` for the 2048 entry.

## Core Game Logic

### Tile Data Model

Each tile is represented by a `Tile` object:

- `id`: unique int, auto-incremented, used as widget key for animations.
- `value`: tile number (2, 4, 8, ...).
- `row`, `col`: current grid position.
- `previousRow`, `previousCol`: position before the last move (for slide animation).
- `mergedFrom`: nullable `List<Tile>` — if this tile was created by a merge, references the two source tiles (for merge animation).
- `isNew`: bool — true on the turn the tile was spawned (for spawn animation). Reset to false after animation completes.

### Board State

- `List<Tile>` flat list of all active tiles (not a 2D array — the 2D grid is derived for logic, but the tile list drives rendering and animation).
- `moveCount`: int, incremented on each valid move.
- Game starts with 2 random tiles.
- New tile generation: 90% chance of 2, 10% chance of 4.

### Slide Algorithm (e.g., slide left)

1. Process each row: collect tiles, merge adjacent equal numbers, assign new positions.
2. **Merge rule**: a tile can only participate in one merge per move. E.g., `[2,2,2,2]` sliding left produces `[4,4,0,0]`, not `[8,0,0,0]`.
3. Accumulate score from merges (merged value added to score).
4. Only spawn a new tile if the board state changed.
5. Set `previousRow`/`previousCol` on moved tiles for animation.

### Undo System

- History stack, max 3 entries. Each entry snapshots: full tile list (deep copy) + score + moveCount.
- Push current state **before** each valid move (before move execution and before new tile spawn).
- Undo pops the top entry and restores the complete pre-move state (the newly spawned tile is also undone).
- 3 undo chances per game, non-replenishable.

### Game Over Detection

- No empty cells AND no adjacent equal cells in any direction → game over.

### Reaching 2048

- Show congratulations dialog with two options:
  - **"Continue"**: enter endless mode, game continues with no upper limit on tile values.
  - **"View Score"**: ends the current game, navigates to `GameOverPage` with final stats. High score is saved.
- While the dialog is showing, the game is considered paused (input ignored). The dialog is displayed as a custom overlay within the game widget (not `showDialog`) to avoid conflicts with `GameScaffold`'s pause overlay and back-button handling.
- The dialog is only shown once per game session.

### Score System

- Each merge adds the merged value (two 16s merge → +32).
- High scores persisted via `ScoreService` with key format: `highscore_2048_4x4`.
- Best score in `GameScaffold` top bar is loaded once at session start and remains static during gameplay (matches Snake behavior).

## Animations

All animations use Flutter's `AnimationController`. Define a shared constant `kTileAnimationDuration = 200ms`.

| Animation | Effect |
|-----------|--------|
| Slide | Tween position interpolation from `previousRow/Col` to `row/col` |
| Merge | Scale up to 1.2× then bounce back (ScaleTransition) |
| New tile | Scale from 0 to 1 + fade in (ScaleTransition + FadeTransition) |

- All animations run in parallel.
- Input is locked during animation to prevent state corruption.

## Input Handling

- **Keyboard**: Arrow keys + WASD for movement, Escape for pause.
- **Swipe**: GestureDetector onPanEnd, direction determined by velocity vector.
- **Animation lock**: Ignore input while animations are in progress.

## UI Layout

- Reuse `GameScaffold` for title, score display, pause functionality.
- Board centered with `AspectRatio(1.0)` to maintain square shape.
- Board background: rounded rectangle `#bbada0`, cell spacing adapts to gridSize:
  - 3×3: 10px, 4×4: 8px, 5×5: 6px, 6×6: 4px.
- Undo button below the board, showing remaining undo count.
- Tile text uses `FittedBox` to auto-scale font size for large numbers (especially important for 5×5/6×6 grids and endless mode values like 16384+).

## Mode Selection Page

Follows `snake_mode_page.dart` pattern:

- Four mode cards with grid icon + name + description.
- Tap navigates to `/2048/play` with `gridSize` argument.

## Game Over Flow

1. **Reach 2048** → Custom overlay: "Congratulations!", buttons: "Continue" / "View Score".
   - "View Score" ends the game and navigates to `GameOverPage`.
2. **No moves left** → Navigate to `GameOverPage` with stats:
   - Score
   - Max tile value
   - Move count
   - Mode (e.g., "4×4")
3. `GameOverPage` provides "Play Again" and "Home" buttons (reused from shared).
4. "Play Again" recreates the game page with `UniqueKey()` (matching Snake pattern).

## Shared Infrastructure Reuse

- `GameScaffold`: top bar, score, pause overlay.
- `ScoreService`: high score persistence.
- `GameOverPage`: game over screen with stats and replay.
